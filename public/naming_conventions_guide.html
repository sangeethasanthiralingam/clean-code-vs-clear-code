Programming Naming Conventions

Learn how to name your variables, functions, and files properly in programming

camelCase

+
Start with lowercase, then capitalize each new word

Examples

myAge = 25
getUserName()
isLoggedIn = true

When to Use

  • Variables (storing data): myName, userAge
  • Functions (actions): getName(), saveData()
  • Most common in JavaScript, Java, C#

Why It's Good

📖 Easy to read
⚡ Quick to type
👍 Most popular
🔧 Editor helps

PascalCase

+
Capitalize the first letter of every word

Examples

class Person
class BankAccount
MyComponent

When to Use

  • Class names (like Person, Car, Database)
  • Components in React (MyButton, UserProfile)
  • Important things that need to stand out

Why It's Good

👑 Looks important
🏗️ Shows structure
📖 Easy to spot
🎨 Standard for classes

snake_case

+
All lowercase letters with underscores between words

Examples

user_name = "John"
get_user_data()
is_logged_in = True

When to Use

  • Python programming (most common)
  • Database table and column names
  • File names (my_script.py)

Why It's Good

🐍 Python loves it
📊 Database friendly
👁️ Very readable
💻 Unix style

SCREAMING_SNAKE_CASE

+
All UPPERCASE letters with underscores between words

Examples

MAX_USERS = 100
API_KEY = "abc123"
DEFAULT_TIMEOUT = 30

When to Use

  • Constants (values that never change)
  • Configuration settings
  • Environment variables

Why It's Good

🚨 Very noticeable
⚠️ Shows "don't change"
🌐 Everyone uses it
🔍 Easy to find

kebab-case

+
All lowercase letters with dashes between words

Examples

<div class="user-profile">
my-website.com/about-us
--main-color: blue;

When to Use

  • CSS class names (.my-button, .user-card)
  • Website URLs (example.com/about-us)
  • HTML attributes and IDs

Why It's Good

🌐 Web standard
🔗 URL friendly
🎨 CSS loves it
📱 SEO friendly

UPPER-KEBAB-CASE

+
All UPPERCASE letters with dashes between words

Examples

Content-Type: application/json
User-Agent: MyBrowser
X-API-KEY: secret123

When to Use

  • HTTP headers (Content-Type, User-Agent)
  • API response headers
  • Some configuration files

Why It's Good

🌐 HTTP standard
📡 API friendly
🔍 Easy to parse
🛠️ Tool support

Quick Guide by Programming Language

JavaScript

  • Variables: userName (camelCase)
  • Functions: getName() (camelCase)
  • Classes: User (PascalCase)
  • Constants: MAX_USERS (SCREAMING_SNAKE_CASE)

Python

  • Variables: user_name (snake_case)
  • Functions: get_name() (snake_case)
  • Classes: User (PascalCase)
  • Constants: MAX_USERS (SCREAMING_SNAKE_CASE)

Java/C#

  • Variables: userName (camelCase)
  • Methods: getName() (camelCase)
  • Classes: User (PascalCase)
  • Constants: MAX_USERS (SCREAMING_SNAKE_CASE)

CSS/HTML

  • Classes: .user-card (kebab-case)
  • IDs: #main-content (kebab-case)
  • Custom properties: --main-color (kebab-case)

Databases

  • Table names: user_profiles (snake_case)
  • Column names: first_name (snake_case)
  • Indexes: idx_user_email (snake_case)

URLs & Files

  • URL paths: /about-us (kebab-case)
  • File names: my-script.js (kebab-case)
  • Python files: my_script.py (snake_case)

💡 Simple Rule to Remember

Pick one style and stick with it in your project! Most code editors will help you follow the right naming rules automatically. When in doubt, look at examples in your programming language's documentation.
